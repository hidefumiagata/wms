# レポート設計書 — 在庫推移レポート

## 1. レポート概要

| 項目 | 内容 |
|------|------|
| **レポートID** | `RPT-08` |
| **レポート名** | 在庫推移レポート |
| **用途** | 倉庫管理者が、指定期間における特定商品の在庫変動履歴（入荷・出荷・移動・訂正・ばらし等）を時系列で確認するために使用する。在庫差異の原因調査、入出庫実績の分析に利用。 |
| **対応API** | `API-RPT-008`（[API-10-report.md](API-10-report.md) を参照） |
| **用紙サイズ** | A4 |
| **向き** | 横（Landscape） |
| **呼び出し元画面** | INV-001（在庫一覧画面）から出力ボタンで呼び出し |

> 入力パラメータ（検索条件）は [API-10-report.md](API-10-report.md) の API-RPT-008 を参照。

## 2. PDFレイアウト

### 2.1. ヘッダー（1ページ目のみ追加表示する情報がある場合）

1ページ目のみ、対象商品の基本情報を追加表示する。

| 要素 | 内容 |
|------|------|
| 対象商品 | 商品コード + 商品名（例: `P-001 商品A`） |
| 対象期間 | `期間: 2026-03-01 ~ 2026-03-17` |

### 2.2. 明細テーブル カラム定義

| # | カラム名 | APIフィールド | 幅(mm) | 配置 | 書式 | 備考 |
|---|---------|-------------|--------|------|------|------|
| 1 | No. | （行番号） | 8 | 右 | 整数 | 通し番号 |
| 2 | 変動日 | `movementDate` | 25 | 中 | 日付 | `yyyy-MM-dd` |
| 3 | 変動種別 | `movementTypeLabel` | 30 | 左 | 文字列 | 日本語ラベル表示 |
| 4 | ロケーションコード | `locationCode` | 38 | 左 | 文字列 | |
| 5 | 荷姿 | `unitType` | 15 | 中 | 文字列 | CAS/BAL/PCS表示 |
| 6 | ロット番号 | `lotNumber` | 30 | 左 | 文字列 | ロット管理OFF時は「—」 |
| 7 | 変動前数量 | `quantityBefore` | 25 | 右 | 整数 | 3桁カンマ区切り |
| 8 | 変動数 | `quantityChange` | 25 | 右 | 整数 | 3桁カンマ。増加は正、減少は負 |
| 9 | 変動後数量 | `quantityAfter` | 25 | 右 | 整数 | 3桁カンマ区切り |
| 10 | 参照番号 | `referenceNumber` | 56 | 左 | 文字列 | 入荷伝票番号・出荷伝票番号等 |

> 幅合計: 8 + 25 + 30 + 38 + 15 + 30 + 25 + 25 + 25 + 56 = **277mm**（A4横印字可能幅に収まる）

### 2.3. グルーピング・小計

```
グルーピングキー: movementDate（変動日）
グループヘッダー: 変動日を太字で1行表示（例: 「2026-03-10」）
小計行: 変動数の日次合計を表示（日ごとの純増減）
ページブレーク: グループ単位で改ページしない（グループ途中で改ページ可）
```

### 2.4. 合計行（テーブル末尾）

| 表示位置 | 内容 |
|---------|------|
| No.〜ロケーションコード列 | 「期間合計」ラベル（太字） |
| 変動数列 | 全行の変動数合計（期間中の純増減） |

### 2.5. ページブレークルール

| ルール | 内容 |
|--------|------|
| 行数制限 | 40行/ページ（超過時に自動改ページ） |
| グループ改ページ | しない |
| 孤立行防止 | グループの最終行が次ページに1行だけ送られる場合、グループごと改ページする |

## 4. 特記事項

- 変動種別（`movementType`）の日本語ラベル対応は以下の通り:
  | movementType | 表示ラベル |
  |-------------|-----------|
  | `INBOUND` | 入庫 |
  | `OUTBOUND` | 出庫 |
  | `MOVE_OUT` | 在庫移動出 |
  | `MOVE_IN` | 在庫移動入 |
  | `BREAKDOWN_OUT` | ばらし出 |
  | `BREAKDOWN_IN` | ばらし入 |
  | `CORRECTION` | 在庫訂正 |
  | `STOCKTAKE_ADJUSTMENT` | 棚卸調整 |
  | `INBOUND_CANCEL` | 入荷キャンセル |
- 変動数が正（増加）の場合は黒字、負（減少）の場合は赤字（`#FF0000`）で表示する。
- 変動種別が `CORRECTION`（在庫訂正）の行は背景色を薄い黄色（`#FFFFF0`）にして視認性を高める。
- ロット管理フラグOFFの商品はロット番号列を「—」で表示する。
- `dateFrom` 未指定時は当月1日、`dateTo` 未指定時は本日がデフォルト（API側で制御）。
- データ取得仕様・フィルタ条件マッピングは [API-10-report.md](API-10-report.md) の API-RPT-008 業務ロジックを参照。
- データ0件の場合は「該当データがありません」メッセージを中央表示する。
- 2ヶ月以上前のデータはバックアップ対象のため参照できない場合がある旨を、該当時にヘッダー条件行の下に注記表示する。
