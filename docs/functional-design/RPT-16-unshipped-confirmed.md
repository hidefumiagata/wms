# レポート設計書 — 未出荷リスト（確定）

## 1. レポート概要

| 項目 | 内容 |
|------|------|
| **レポートID** | `RPT-16` |
| **レポート名** | 未出荷リスト（確定） |
| **用途** | 倉庫管理者・倉庫スタッフが、日替処理で確定した営業日末時点の未出荷伝票を確認するために使用する。出荷遅延の把握、翌日の出荷優先順位決定、出荷先への遅延連絡の判断材料とする |
| **対応API** | `API-RPT-016`（[API-10-report.md](API-10-report.md) を参照） |
| **用紙サイズ** | A4 |
| **向き** | 横（Landscape） |
| **呼び出し元画面** | OUT-001（受注一覧画面）から「未出荷リスト（確定）」ボタンで呼び出し |

> 入力パラメータ（検索条件）は [API-10-report.md](API-10-report.md) の API-RPT-016 を参照。

## 2. PDFレイアウト

### 2.1. ヘッダー（1ページ目のみ追加表示する情報）

共通ヘッダー（[_standard-report.md](_standard-report.md) 参照）に加え、以下の情報を1ページ目に表示する。

| 要素 | 位置 | 内容 |
|------|------|------|
| バッチ処理営業日 | 左 | `batchBusinessDate`（例: `2026-03-14`） |
| 対象件数 | 右 | 「対象: {N}件」（明細行の総件数） |

> 抽出条件サマリーには「バッチ処理営業日: 2026-03-14」を表示する。

### 2.2. 明細テーブル カラム定義

| # | カラム名 | APIフィールド | 幅(mm) | 配置 | 書式 | 備考 |
|---|---------|-------------|--------|------|------|------|
| 1 | No. | （行番号） | 10 | 右 | 整数 | 自動採番 |
| 2 | 伝票番号 | `slipNumber` | 40 | 左 | — | |
| 3 | 出荷先名 | `customerName` | 55 | 左 | — | 長い場合は折り返し |
| 4 | 出荷予定日 | `plannedShipDate` | 28 | 中 | 日付 | |
| 5 | 商品コード | `productCode` | 30 | 左 | — | |
| 6 | 商品名 | `productName` | 55 | 左 | — | 長い場合は折り返し |
| 7 | 数量(ケース) | `totalQuantityCas` | 25 | 右 | 整数 | |
| 8 | ステータス | `statusAtBatch` | 34 | 中 | — | ステータスラベルで表示 |

> 幅合計: 10 + 40 + 55 + 28 + 30 + 55 + 25 + 34 = **277mm**（A4横の印字可能幅 277mm 以内）

### 2.3. グルーピング・小計

```
グルーピングキー: slipNumber（出荷伝票番号）
小計行: 数量(ケース)の合計を表示
グループヘッダー: なし（伝票番号は各行に表示。同一伝票の2行目以降は伝票番号・出荷先名・出荷予定日・ステータスを省略表示「〃」）
ページブレーク: グループ単位で改ページしない
```

### 2.4. 合計行（テーブル末尾）

| 表示位置 | 内容 |
|---------|------|
| 商品名列 | 「合計」ラベル |
| 数量(ケース)列 | 全行の `totalQuantityCas` 合計値 |

### 2.5. ページブレークルール

| ルール | 内容 |
|--------|------|
| 行数制限 | 30行/ページ（超過時に自動改ページ） |
| グループ改ページ | しない |
| 孤立行防止 | 同一伝票番号の最終行が次ページに1行だけ送られる場合、グループごと改ページする |

## 4. 特記事項

- **RPT-15（リアルタイム版）との関係**: 本レポート（RPT-16）はRPT-15（未出荷リスト・リアルタイム）と類似の目的を持つが、レイアウトは異なる。RPT-15は出荷先別グルーピング・遅延日数・遅延度列を含む10列構成であるのに対し、RPT-16は伝票番号別グルーピングの8列構成である。主な違いはデータ取得タイミングにある。RPT-15は `outbound_slips` テーブルをリアルタイムに参照するのに対し、RPT-16は日替処理で `unshipped_list_records` テーブルに確定されたスナップショットデータを参照する
- **データ取得元**: `unshipped_list_records` テーブル（日替処理で生成）。データ取得仕様の詳細は [API-10-report.md](API-10-report.md) の API-RPT-016 業務ロジックを参照
- **ステータス表示**: `statusAtBatch` フィールドのステータスコードを以下のラベルに変換して表示する
  - `ORDERED` → 受注
  - `PARTIAL_ALLOCATED` → 一部引当
  - `ALLOCATED` → 引当完了
  - `PICKING_COMPLETED` → ピッキング完了
  - `INSPECTING` → 出荷検品中
- **遅延行の強調表示**: 出荷予定日がバッチ処理営業日より前の行（遅延している行）は、行全体の背景色を薄い赤（`#FFF1F1`）で強調表示する
- **同一伝票の省略表示**: 同一伝票番号の2行目以降は、伝票番号・出荷先名・出荷予定日・ステータスを「〃」で省略表示し、視認性を向上させる
- **データ0件**: 指定営業日に未出荷データが存在しない場合、明細テーブル領域に「該当データがありません」メッセージを中央表示する
- **ソート順**: 出荷予定日（ASC）→ 伝票番号（ASC）→ 商品コード（ASC）の順でソートする
- **テンプレートファイル配置**: `backend/src/main/resources/templates/reports/RPT-16.html`
