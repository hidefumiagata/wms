# レポート設計書 — 入庫実績レポート

## 1. レポート概要

| 項目 | 内容 |
|------|------|
| **レポートID** | `RPT-04` |
| **レポート名** | 入庫実績レポート |
| **用途** | 倉庫管理者が、指定期間内に入庫完了した実績を仕入先別に確認し、入荷精度の分析・仕入先への報告・在庫配置状況の把握に活用する。 |
| **対応API** | `API-RPT-004`（[API-10-report.md](API-10-report.md) を参照） |
| **用紙サイズ** | A4 |
| **向き** | 横（Landscape） |
| **呼び出し元画面** | INB-006（入庫実績照会画面）から出力ボタンで呼び出し |

> 入力パラメータ（検索条件）は [API-10-report.md](API-10-report.md) の API-RPT-004 を参照。

```
グルーピングキー: supplierName（仕入先名）
```

## 4. 特記事項

- 差異がある行（`diffQuantityCas != 0`）は背景色をピンク（`#FFF1F1`）で強調表示する
- 差異列の値が正の場合は `+` 符号を付ける（例: `+2`）、負の場合はマイナス符号をそのまま表示（例: `-1`）、ゼロの場合は `0` と表示
- 対象データは入庫完了（`status = 'STORED'`）のレコードのみ。詳細は API-RPT-004 の業務ロジックを参照
- データ0件の場合は「該当データがありません」メッセージを中央表示する
- **返品数量**: 返品数量は入荷返品（`return_type='INBOUND'`）のうち、当該入荷伝票に紐づくもののみ表示する。返品がない場合は「—」（emダッシュ）で表示
- ソート順は仕入先名（昇順）→ 入庫日（昇順）→ 伝票番号（昇順）。詳細は API-RPT-004 のデータ取得仕様を参照
