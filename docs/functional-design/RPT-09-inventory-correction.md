# レポート設計書 — 在庫訂正一覧

## 1. レポート概要

| 項目 | 内容 |
|------|------|
| **レポートID** | `RPT-09` |
| **レポート名** | 在庫訂正一覧 |
| **用途** | 倉庫管理者・監査担当者が、指定倉庫・期間における在庫訂正の実施履歴を確認するために使用する。監査・内部統制の証跡として、訂正前後の数量・差異・訂正理由・実施者を一覧で把握し、不正や誤操作の検知に活用する |
| **対応API** | `API-RPT-009`（[API-10-report.md](API-10-report.md) を参照） |
| **用紙サイズ** | A4 |
| **向き** | 横（Landscape） |
| **呼び出し元画面** | INV-004（在庫訂正登録）から「訂正一覧レポート」ボタンで呼び出し |

> 入力パラメータ（検索条件）は [API-10-report.md](API-10-report.md) の API-RPT-009 を参照。

## 4. 特記事項

- **行の条件付き背景色**: 変動数（`quantityChange`）の正負で行背景色を切り替える
  - 増加訂正（`quantityChange > 0`）: 薄青（`#F0F4FF`）
  - 減少訂正（`quantityChange < 0`）: 薄赤（`#FFF1F1`）
  - 変動なし（`quantityChange = 0`）: 背景色なし（白）
- **変動数の符号表示**: 正値には `+` を明示的に付与する（例: `+5`）。負値はそのまま表示（例: `-3`）
- **監査証跡としての重要性**: 本レポートは監査・内部統制の証跡として使用されるため、実施者（`operatorName`）の表示は必須。APIレスポンスの `operatorName` はPDFでは出力者とは別に各行で表示されるが、カラム幅の制約から明細テーブルには含めず、訂正理由列の末尾に `（実施者名）` を括弧付きで追記する形式とする
- **データ0件の場合**: 「該当する在庫訂正データがありません」メッセージを明細テーブル部分の中央に表示する
- **ソート順**: 訂正日（`correctionDate`）昇順。同一日内はロケーションコード昇順
- **データ取得仕様**: [API-10-report.md](API-10-report.md) の API-RPT-009 業務ロジックを参照
