# レポート設計書 — 入荷予定レポート

## 1. レポート概要

| 項目 | 内容 |
|------|------|
| **レポートID** | `RPT-03` |
| **レポート名** | 入荷予定レポート |
| **用途** | 倉庫管理者・現場担当者が、指定期間内の入荷予定を仕入先別に把握し、入荷作業の計画・人員配置・保管場所の準備に活用する。 |
| **対応API** | `API-RPT-003`（[API-10-report.md](API-10-report.md) を参照） |
| **用紙サイズ** | A4 |
| **向き** | 横（Landscape） |
| **呼び出し元画面** | INB-001（入荷予定一覧画面）から出力ボタンで呼び出し |

> 入力パラメータ（検索条件）は [API-10-report.md](API-10-report.md) の API-RPT-003 を参照。

## 2. PDFレイアウト

### 2.1. ヘッダー（1ページ目のみ追加表示する情報がある場合）

共通ヘッダーのみ（_standard-report.md 参照）

抽出条件サマリー例: `期間: 2026-03-01 ～ 2026-03-31 / ステータス: 入荷予定`

### 2.2. 明細テーブル カラム定義

| # | カラム名 | APIフィールド | 幅(mm) | 配置 | 書式 | 備考 |
|---|---------|-------------|--------|------|------|------|
| 1 | No. | （行番号） | 8 | 右 | 整数 | グループ内通し番号 |
| 2 | 伝票番号 | `slipNumber` | 40 | 左 | — | |
| 3 | 入荷予定日 | `plannedDate` | 28 | 中 | `yyyy-MM-dd` | |
| 4 | 商品コード | `productCode` | 30 | 左 | — | |
| 5 | 商品名 | `productName` | 65 | 左 | — | 長い場合は末尾省略 |
| 6 | 予定数(ケース) | `plannedQuantityCas` | 28 | 右 | 整数 | 3桁カンマ区切り |
| 7 | 予定数(バラ) | `plannedQuantityPcs` | 28 | 右 | 整数 | 3桁カンマ区切り |
| 8 | ステータス | `statusLabel` | 30 | 中 | — | 日本語表示名 |
| | **合計** | | **257** | | | A4横277mm以内 |

> 幅の合計は用紙の印字可能幅（A4横: 約277mm）以内に収めている。残余20mmはカラム間パディングに使用。

### 2.3. グルーピング・小計

```
グルーピングキー: supplierName（仕入先名）
グループヘッダー: 「仕入先: {supplierName}」を太字で表示（明細テーブルの上部に1行）
小計行: 予定数(ケース)、予定数(バラ) の合計をグループ末尾に表示
ページブレーク: グループ単位で改ページしない（連続印刷）
```

### 2.4. 合計行（テーブル末尾）

| 表示位置 | 内容 |
|---------|------|
| 商品名列 | 「合計」ラベル（太字） |
| 予定数(ケース)列 | 全行の予定数(ケース)合計値 |
| 予定数(バラ)列 | 全行の予定数(バラ)合計値 |

### 2.5. ページブレークルール

| ルール | 内容 |
|--------|------|
| 行数制限 | 35行/ページ（超過時に自動改ページ） |
| グループ改ページ | しない |
| 孤立行防止 | グループの最終行が次ページに1行だけ送られる場合、グループごと改ページする |

## 4. 特記事項

- ステータスが `CANCELLED`（キャンセル）の行は取り消し線スタイル（テキスト色: `#999999`）で表示する
- ステータスが `STORED`（入庫完了）の行は背景色をライトグリーン（`#F0FFF0`）で表示する
- データ0件の場合は「該当データがありません」メッセージを中央表示する
- ソート順は仕入先名（昇順）→ 入荷予定日（昇順）→ 伝票番号（昇順）。詳細は API-RPT-003 のデータ取得仕様を参照
