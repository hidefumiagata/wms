# レポート設計書 — 未入荷リスト（リアルタイム）

## 1. レポート概要

| 項目 | 内容 |
|------|------|
| **レポートID** | `RPT-05` |
| **レポート名** | 未入荷リスト（リアルタイム） |
| **用途** | 倉庫担当者・管理者が、入荷予定日を過ぎてもまだ入荷完了していない伝票・明細をリアルタイムで確認し、仕入先への督促や入荷作業の優先度判断に使用する |
| **対応API** | `API-RPT-005`（[API-10-report.md](API-10-report.md) を参照） |
| **用紙サイズ** | A4 |
| **向き** | 横（Landscape） |
| **呼び出し元画面** | INB-001（入荷一覧画面）から出力ボタンで呼び出し |

> 入力パラメータ（検索条件）は [API-10-report.md](API-10-report.md) の API-RPT-005 を参照。

## 2. PDFレイアウト

### 2.1. ヘッダー（1ページ目のみ追加表示する情報がある場合）

共通ヘッダーに加え、以下を表示する。

### 2.2. 明細テーブル カラム定義

| # | カラム名 | APIフィールド | 幅(mm) | 配置 | 書式 | 備考 |

| 1 | No. | （行番号） | 10 | 右 | 整数 | グループ内通番 |
| 2 | 伝票番号 | `slipNumber` | 40 | 左 | — | |
| 3 | 入荷予定日 | `plannedDate` | 28 | 中 | `yyyy-MM-dd` | |
| 4 | 遅延日数 | `delayDays` | 20 | 右 | 整数 + 「日」 | 例: `5日` |
| 5 | 商品コード | `productCode` | 30 | 左 | — | |
| 6 | 商品名 | `productName` | 60 | 左 | — | 長い場合は末尾省略 |
| 7 | 予定数(ケース) | `plannedQuantityCas` | 25 | 右 | 整数（3桁カンマ） | |
| 8 | ステータス | `statusLabel` | 30 | 中 | — | 日本語ラベル表示 |

> 幅合計: 10 + 40 + 28 + 20 + 30 + 60 + 25 + 30 + 34 = **277mm**（A4横の印字可能幅以内）

### 2.3. グルーピング・小計

```
グルーピングキー: supplierName（仕入先名）
グループヘッダー: 仕入先名を太字で1行表示（例: 「■ テスト仕入先B」）
小計行: グループ末尾に「予定数(ケース)」列の小計、および明細件数を表示
ページブレーク: グループ単位で改ページしない（グループ途中でのページ跨ぎを許容）
```

### 2.4. 合計行（テーブル末尾）

| 表示位置 | 内容 |
|---------|------|
| 商品名列 | 「合計」ラベル |
| 予定数(ケース)列 | 全明細行の予定数合計 |

加えて、総明細件数を合計行に表示する（例: 「合計 15件」）。

### 2.5. ページブレークルール

| ルール | 内容 |
|--------|------|
| 行数制限 | 35行/ページ（超過時に自動改ページ） |
| グループ改ページ | しない |
| 孤立行防止 | グループヘッダーが最終行に来る場合は改ページし、次ページ先頭から開始する |

## 4. 特記事項

- **リアルタイムデータ取得**: 本レポートは `inbound_slips` テーブルをリアルタイムに参照するため、出力タイミングによって結果が変動する。確定時点のスナップショットが必要な場合は RPT-06（未入荷リスト（確定））を使用すること。
- **ソート順**: 仕入先名（昇順） → 入荷予定日（昇順。予定日が古い＝遅延が大きい順が上位）で出力する。データ取得仕様の詳細は API-RPT-005 の業務ロジックを参照。
- **遅延日数の強調表示**: 遅延日数が7日以上の行は背景色をピンク（`#FFF1F1`）にして視覚的に強調する。
- **データ0件の場合**: 明細テーブルの代わりに「該当する未入荷データはありません」メッセージを中央表示する。
- **件数上限**: API仕様により最大10,000件。上限超過時はPDFヘッダーの条件行に「※ 10,000件を超過したため一部のみ表示」と注記する。
- **キャンセル・入庫完了済除外**: ステータスが `STORED`（入庫完了）および `CANCELLED`（キャンセル）の伝票は対象外。対象ステータスの詳細は API-RPT-005 の業務ロジックを参照。
