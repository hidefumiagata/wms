# レポート設計書 — 返品レポート

## 1. レポート概要

| 項目 | 内容 |
|------|------|
| **レポートID** | `RPT-18` |
| **レポート名** | 返品レポート |
| **用途** | 倉庫管理者・品質管理担当者が、指定倉庫・期間における返品実績を返品種別ごとに一覧照会し、返品理由の分析・仕入先への報告・返品率の把握に活用する |
| **対応API** | `API-RPT-018`（[API-10-report.md](API-10-report.md) を参照） |
| **用紙サイズ** | A4 |
| **向き** | 横（Landscape） |
| **呼び出し元画面** | RTN-001（返品登録画面）からレポート出力ボタンで呼び出し |

> 入力パラメータ（検索条件）は [API-10-report.md](API-10-report.md) の API-RPT-018 を参照。

## 2. PDFレイアウト

### 2.1. ヘッダー（1ページ目のみ追加表示する情報がある場合）

共通ヘッダーのみ（[_standard-report.md](_standard-report.md) 参照）。

抽出条件サマリー表示例: `期間: 2026-03-01 〜 2026-03-31 / 返品種別: 入荷返品 / 仕入先: テスト仕入先A`

### 2.2. 明細テーブル カラム定義

| # | カラム名 | APIフィールド | 幅(mm) | 配置 | 書式 | 備考 |
|---|---------|-------------|--------|------|------|------|
| 1 | No. | （行番号） | 8 | 右 | 整数 | グループ内通し番号 |
| 2 | 返品伝票番号 | `returnNumber` | 35 | 左 | — | |
| 3 | 返品日 | `returnDate` | 25 | 中 | `yyyy-MM-dd` | |
| 4 | 商品コード | `productCode` | 25 | 左 | — | |
| 5 | 商品名 | `productName` | 45 | 左 | — | 長い場合は末尾省略（`...`） |
| 6 | 数量 | `quantity` | 20 | 右 | 3桁カンマ | |
| 7 | 荷姿 | `unitType` | 15 | 中 | — | CAS / BAL / PCS |
| 8 | 返品理由 | `returnReasonLabel` | 35 | 左 | — | 長い場合は末尾省略（`...`） |
| 9 | 関連伝票番号 | `relatedSlipNumber` | 35 | 左 | — | 入荷伝票番号 or 出荷伝票番号 |
| 10 | 仕入先名 | `partnerName` | 34 | 左 | — | 長い場合は末尾省略（`...`） |
| | **合計** | | **277** | | | A4横の印字可能幅以内 |

> 幅合計: 8 + 35 + 25 + 25 + 45 + 20 + 15 + 35 + 35 + 34 = **277mm**（A4横の印字可能幅以内）

### 2.3. グルーピング・小計

```
グルーピングキー: returnTypeLabel（返品種別）
グループヘッダー: 「返品種別: {returnTypeLabel}」を太字で表示（明細テーブルの上部に1行）
小計行: 数量（quantity）の合計をグループ末尾に表示
ページブレーク: グループ単位で改ページしない（連続印刷）
```

### 2.4. 合計行（テーブル末尾）

| 表示位置 | 内容 |
|---------|------|
| 商品名列 | 「合計」ラベル（太字） |
| 数量列 | 全行の数量合計値 |

### 2.5. ページブレークルール

| ルール | 内容 |
|--------|------|
| 行数制限 | 35行/ページ（超過時に自動改ページ） |
| グループ改ページ | しない |
| 孤立行防止 | グループの最終行が次ページに1行だけ送られる場合、グループごと改ページする |

## 4. 特記事項

- **ソート順**: 返品種別（`returnTypeLabel`）昇順 → 返品日（`returnDate`）昇順 → 返品伝票番号（`returnNumber`）昇順。詳細は API-RPT-018 のデータ取得仕様を参照
- **関連伝票番号の表示**: 入荷返品の場合は入荷伝票番号、出荷返品の場合は出荷伝票番号を表示する。関連伝票がない場合は「—」（emダッシュ）で表示
- **仕入先名の表示**: 入荷返品の場合は入荷伝票の仕入先名、出荷返品の場合は出荷伝票の届け先パートナー名を表示する
- **仕入先名が未指定の場合**: 取引先（仕入先）が未指定の返品伝票の場合、仕入先名カラムは空欄（`—`）で表示する
- **データ0件の場合**: 「該当する返品データがありません」メッセージを明細テーブル部分の中央に表示する
- **データ取得仕様**: [API-10-report.md](API-10-report.md) の API-RPT-018 業務ロジックを参照
- **テンプレートファイル配置**: `backend/src/main/resources/templates/reports/RPT-18.html`
