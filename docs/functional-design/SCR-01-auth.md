## AUTH-001 ログイン

### 1. 画面概要

| 項目 | 内容 |
|------|------|
| **画面ID** | `AUTH-001` |
| **画面名** | ログイン |
| **URL パス** | `/login` |
| **対象ロール** | なし（未認証） |
| **概要** | WMSのエントリポイント。ユーザーコードとパスワードを入力して認証を行う。認証成功後、`passwordChangeRequired=true` の場合はパスワード変更画面（AUTH-002）へ、`false` の場合はメインメニュー（`/`）へ遷移する。 |

### 4. イベント一覧

| イベントID | トリガー | 処理概要 | 遷移先 / 結果 | 実行可能ロール |
|------------|---------|----------|--------------|--------------|
| `EVT-AUTH001-001` | 画面表示（マウント時） | 認証済みかどうかをフロントエンドストア（Pinia）で確認。認証済みの場合はメインメニューへリダイレクト。クエリパラメータ `reason=session_expired` が存在する場合はセッションタイムアウトメッセージ（`MSG-I-AUTH001-001`）を表示 | 認証済み → `/`（メインメニュー） | — |
| `EVT-AUTH001-002` | 「ログイン」ボタンクリック | (1) フォームバリデーション実行。エラーがあれば送信を中断し項目下にエラーメッセージ表示。(2) バリデーション通過後 `POST /api/v1/auth/login` を呼び出し。(3) ボタンをローディング状態に変更（二重送信防止） | バリデーションエラー → 同画面（エラー表示）、API成功 → `EVT-AUTH001-003` へ、API失敗 → 同画面（エラーバナー表示） | — |
| `EVT-AUTH001-003` | ログインAPI成功（200 OK） | レスポンスのユーザー情報をPiniaストア（authStore）に保存。`passwordChangeRequired` フラグを確認し、遷移先を決定 | `passwordChangeRequired=true` → `/change-password`（AUTH-002）、`false` → `/`（メインメニュー） | — |
| `EVT-AUTH001-004` | 「パスワードを忘れた方」リンククリック | `/auth/reset-request`（AUTH-003）へ遷移 | `/auth/reset-request` | — |

**APIリクエスト仕様（EVT-AUTH001-002）**

```
POST /api/v1/auth/login
Content-Type: application/json

{
  "userCode": "yamada.taro",
  "password": "password123"
}
```

レスポンス:
- `200 OK`: 認証成功。レスポンスボディにユーザー情報（`userId`, `userCode`, `fullName`, `role`, `passwordChangeRequired`）を含む。`access_token` および `refresh_token` が httpOnly Cookie にセットされる。詳細は [API-01-auth.md](API-01-auth.md) の API-AUTH-001 を参照
- `401 Unauthorized`（`INVALID_CREDENTIALS`）: ユーザーコード不存在またはパスワード不一致 → `MSG-E-AUTH001-003` を表示
- `401 Unauthorized`（`ACCOUNT_LOCKED`）: アカウントがロックされている → `MSG-E-AUTH001-002` を表示
- `401 Unauthorized`（`ACCOUNT_INACTIVE`）: アカウントが無効化されている → `MSG-E-AUTH001-003` を表示（セキュリティ上、無効アカウントの存在を露出しない）

**エラーハンドリングの補足**

- `INVALID_CREDENTIALS` 受信時: API レスポンスに残り試行回数は含まれないため、フロントエンドではエラーメッセージのみ表示する（残り回数の表示はサーバー側の情報漏洩リスクがあるため行わない）
- `ACCOUNT_LOCKED` 受信時: ロックメッセージ（`MSG-E-AUTH001-002`）を表示し、ユーザーに管理者への連絡を促す
- `ACCOUNT_INACTIVE` 受信時: `INVALID_CREDENTIALS` と同一のメッセージを表示する（アカウントの存在・状態を外部に露出させない）

### 5. メッセージ一覧

| メッセージID | 種別 | 発生条件 | メッセージ文（日本語） |
|------------|------|---------|-------------------|
| `MSG-E-AUTH001-001` | エラー | ユーザーコードが未入力 | ユーザーコードは必須です |
| `MSG-E-AUTH001-002` | エラー | パスワードが未入力 | パスワードは必須です |
| `MSG-E-AUTH001-003` | エラー | APIエラー: 認証失敗（401 `INVALID_CREDENTIALS` / `ACCOUNT_INACTIVE`） | ユーザーコードまたはパスワードが正しくありません |
| `MSG-E-AUTH001-004` | エラー | APIエラー: アカウントロック（401 `ACCOUNT_LOCKED`） | アカウントがロックされました。管理者に連絡してください |
| `MSG-E-AUTH001-005` | エラー | APIエラー: 予期しないエラー（500系） | ログインに失敗しました。時間をおいて再度お試しください |
| `MSG-I-AUTH001-001` | 情報 | セッションタイムアウトによるリダイレクト（クエリパラメータ `reason=session_expired`） | セッションがタイムアウトしました。再度ログインしてください |

**メッセージ表示方法**:
- `MSG-E-AUTH001-001` 〜 `MSG-E-AUTH001-002`: フォーム項目直下にインラインエラーとして表示（VeeValidate + Zod によるフォームバリデーション）

- `MSG-I-AUTH001-001`: フォーム上部に情報バナー（`alert-info` スタイル）として表示。画面マウント時にクエリパラメータを確認し、表示後にURLからパラメータを除去する

### 4. イベント一覧

| イベントID | トリガー | 処理概要 | 遷移先 / 結果 | 実行可能ロール |
|------------|---------|----------|--------------|--------------|
| `EVT-AUTH002-001` | 画面表示（マウント時） | ログイン済みかつ `password_change_required = true` であることをフロントエンドストア（Pinia）で確認。未認証または変更不要の場合はリダイレクト | 未認証 → `/login`、変更不要 → `/` | 全ロール |
| `EVT-AUTH002-002` | 「新しいパスワード」フィールド入力中（input イベント） | パスワード強度を即時評価してインジケーターを更新。評価基準：弱＝ポリシー未達（8文字未満、または英大文字・英小文字・数字のいずれかが欠如）、中＝ポリシー達成（8文字以上、英大文字・英小文字・数字を含む）、強＝ポリシー達成＋12文字以上 | 同画面（インジケーター更新） | 全ロール |
| `EVT-AUTH002-003` | 「パスワードを変更する」ボタンクリック | (1) フォームバリデーション実行。エラーがあれば送信を中断し項目下にエラーメッセージ表示。(2) バリデーション通過後 `POST /api/v1/auth/change-password` を呼び出し。(3) ボタンをローディング状態に変更（二重送信防止） | バリデーションエラー → 同画面（エラー表示）、API成功 → `/`（メインメニュー）、API失敗 → 同画面（エラーバナー表示） | 全ロール |
| `EVT-AUTH002-004` | 「ログアウト」リンククリック | `POST /api/v1/auth/logout` を呼び出してセッション破棄（Cookieクリア）。Piniaストアのユーザー情報をリセット | `/login` | 全ロール |

**画面遷移保護（Vue Router ナビゲーションガード）**

- `password_change_required = true` の間は、`/change-password` 以外のルートへのアクセスを `/change-password` にリダイレクトする
- ブラウザバック・URL直接入力も含めて一切の迂回を防止する

**APIリクエスト仕様（EVT-AUTH002-003）**

```
POST /api/v1/auth/change-password
Content-Type: application/json
Cookie: access_token=<JWT>

{
  "currentPassword": "InitialP@ss1",
  "newPassword":     "NewSecure#99"
}
```

レスポンス:
- `204 No Content`: パスワード変更成功（レスポンスボディなし）。バックエンドが `password_change_required` フラグを `false` に更新。フロントエンドは Pinia ストアの `passwordChangeRequired` を `false` にセットしてから `/` へ遷移
- `400 Bad Request`: 現在のパスワード誤り、またはポリシー違反
- `401 Unauthorized`: セッション期限切れ → `/login` へリダイレクト
