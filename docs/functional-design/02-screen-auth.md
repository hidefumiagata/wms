## AUTH-002 初回パスワード変更

### 1. 画面概要

| 項目 | 内容 |
|------|------|
| **画面ID** | `AUTH-002` |
| **画面名** | 初回パスワード変更 |
| **URL パス** | `/change-password` |
| **対象ロール** | 全ロール（SYSTEM_ADMIN / WAREHOUSE_MANAGER / WAREHOUSE_STAFF / VIEWER） |
| **概要** | 初回ログイン時に強制表示される画面。管理者が発行した初期パスワードを新しいパスワードに変更するまで、他の一切の操作・画面遷移は不可。変更完了後はメインメニューへ遷移する。 |

**フォームエリアの詳細**

- 上部に黄色の注意バナー「初回ログインです。パスワードを変更してください。変更が完了するまで他の操作は行えません。」を表示
- 各パスワード入力欄に目のアイコン（表示/非表示トグル）を設置
- 「新しいパスワード」入力後にリアルタイムで強度インジケーターを表示（弱/中/強）
- 送信ボタン下部に「このアカウントでのログインをやめる場合はこちら」リンクを配置（ログアウト処理）

### 4. イベント一覧

| イベントID | トリガー | 処理概要 | 遷移先 / 結果 | 実行可能ロール |
|------------|---------|----------|--------------|--------------|
| `EVT-AUTH002-001` | 画面表示（マウント時） | ログイン済みかつ `password_change_required = true` であることをフロントエンドストア（Pinia）で確認。未認証または変更不要の場合はリダイレクト | 未認証 → `/login`、変更不要 → `/` | 全ロール |
| `EVT-AUTH002-002` | 「新しいパスワード」フィールド入力中（input イベント） | パスワード強度を即時評価してインジケーターを更新。評価基準：4文字未満＝弱、8文字未満＝中、8文字以上＝強（アルファベット＋数字混在で加点） | 同画面（インジケーター更新） | 全ロール |
| `EVT-AUTH002-003` | 「パスワードを変更する」ボタンクリック | (1) フォームバリデーション実行。エラーがあれば送信を中断し項目下にエラーメッセージ表示。(2) バリデーション通過後 `POST /api/v1/auth/change-password` を呼び出し。(3) ボタンをローディング状態に変更（二重送信防止） | バリデーションエラー → 同画面（エラー表示）、API成功 → `/`（メインメニュー）、API失敗 → 同画面（エラーバナー表示） | 全ロール |
| `EVT-AUTH002-004` | 「ログアウト」リンククリック | `POST /api/v1/auth/logout` を呼び出してセッション破棄（Cookieクリア）。Piniaストアのユーザー情報をリセット | `/login` | 全ロール |

**画面遷移保護（Vue Router ナビゲーションガード）**

- `password_change_required = true` の間は、`/change-password` 以外のルートへのアクセスを `/change-password` にリダイレクトする
- ブラウザバック・URL直接入力も含めて一切の迂回を防止する

**APIリクエスト仕様（EVT-AUTH002-003）**

```
POST /api/v1/auth/change-password
Content-Type: application/json
Cookie: access_token=<JWT>

{
  "currentPassword": "InitialP@ss1",
  "newPassword":     "NewSecure#99"
}
```

レスポンス:
- `200 OK`: パスワード変更成功。バックエンドが `password_change_required` フラグを `false` に更新。フロントエンドは Pinia ストアの `passwordChangeRequired` を `false` にセットしてから `/` へ遷移
- `400 Bad Request`: 現在のパスワード誤り、またはポリシー違反
- `401 Unauthorized`: セッション期限切れ → `/login` へリダイレクト

### 5. メッセージ一覧

| メッセージID | 種別 | 発生条件 | メッセージ文（日本語） |
|------------|------|---------|-------------------|
| `MSG-E-010` | エラー | 現在のパスワードが未入力 | 現在のパスワードは必須です |
| `MSG-E-011` | エラー | 新しいパスワードが未入力 | 新しいパスワードは必須です |
| `MSG-E-012` | エラー | 新しいパスワードが8文字未満 | パスワードは8文字以上で入力してください |
| `MSG-E-013` | エラー | 新しいパスワードが現在のパスワードと同一 | 現在のパスワードと異なるパスワードを設定してください |
| `MSG-E-014` | エラー | 新しいパスワード（確認）が未入力 | パスワード（確認）は必須です |
| `MSG-E-015` | エラー | 新しいパスワードと確認が不一致 | パスワードが一致しません。再度入力してください |
| `MSG-E-016` | エラー | APIエラー：現在のパスワード誤り（400） | 現在のパスワードが正しくありません ※パスワード変更APIの連続失敗ロックポリシーは未定義のため、要件確定後に画面仕様を更新すること |
| `MSG-E-017` | エラー | APIエラー：予期しないエラー（500系） | パスワード変更に失敗しました。時間をおいて再度お試しください |
| `MSG-S-001` | 成功 | パスワード変更成功 | パスワードを変更しました。ようこそ、WMSへ。 |

**メッセージ表示方法**:
- `MSG-E-010` 〜 `MSG-E-015`: フォーム項目直下にインラインエラーとして表示（VeeValidate + Zod によるフォームバリデーション）
- `MSG-E-016` 〜 `MSG-E-017`: フォーム上部にエラーバナー（`alert-error` スタイル）として表示
- `MSG-S-001`: `el-message`（type: success）でトースト通知。表示後 `/` へ遷移
