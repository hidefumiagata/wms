## BAT-001 日替処理実行

### 1. 画面概要

| 項目 | 内容 |
|------|------|
| **画面ID** | `BAT-001` |
| **画面名** | 日替処理実行 |
| **URL パス** | `/batch/daily-close` |
| **対象ロール** | SYSTEM_ADMIN, WAREHOUSE_MANAGER |
| **概要** | 対象営業日を指定して日替処理（営業日更新・入荷実績集計・出荷実績集計・在庫集計・トランデータバックアップ・日次集計レコード生成）を手動実行する画面。実行中は6つの処理の進捗をステップ形式でリアルタイム表示し、完了後に結果を確認できる。途中失敗からの再実行時は、完了済みステップを「スキップ（完了済み）」として表示し、未完了のステップから処理を再開する（冪等実行）。 |

### 4. イベント一覧

| イベントID | トリガー | 処理概要 | 遷移先 / 結果 | 実行可能ロール |
|----------|---------|---------|-------------|-------------|
| `EVT-BAT001-001` | 画面初期表示 | 現在営業日をAPIから取得して表示。対象営業日を翌営業日に設定する | 同画面（初期状態） | SYSTEM_ADMIN, WAREHOUSE_MANAGER |
| `EVT-BAT001-002` | 「日替処理実行」ボタンクリック | 確認ダイアログ（MSG-W-BAT001-001）を表示する | 確認ダイアログ | SYSTEM_ADMIN, WAREHOUSE_MANAGER |
| `EVT-BAT001-003` | 確認ダイアログ「実行」クリック | `POST /api/v1/batch/daily-close` を呼び出す。処理進捗カードを表示し、ポーリング（2秒間隔でGET /api/v1/batch/executions/{id}）で各ステップのステータスをリアルタイム更新する。実行ボタンを無効化する。途中失敗からの再実行時は、完了済みステップをスキップし、未完了のステップから再開する。営業日更新（Step1）は冪等に実行される（既に更新済みの場合はスキップ）。スキップされたステップにはMSG-I-BAT001-001を表示する。サーバー側で冪等性を制御するため、APIエンドポイントは同一（`POST /api/v1/batch/daily-close`） | 同画面（実行中状態） | SYSTEM_ADMIN, WAREHOUSE_MANAGER |
| `EVT-BAT001-004` | バッチ処理完了（成功） | 実行結果カードを表示（成功）。MSG-S-BAT001-001を表示する | 同画面（完了状態） | — |
| `EVT-BAT001-005` | バッチ処理完了（失敗） | 実行結果カードを表示（失敗・エラー内容）。失敗したステップを赤色で強調表示する。MSG-E-BAT001-002を表示する | 同画面（完了状態） | — |
| `EVT-BAT001-006` | 処理済み営業日を指定して実行ボタンクリック | APIから二重実行エラーを受信。MSG-E-BAT001-001を表示する | 同画面（エラー表示） | SYSTEM_ADMIN, WAREHOUSE_MANAGER |

### 5. メッセージ一覧

| メッセージID | 種別 | 発生条件 | メッセージ文（日本語） |
|------------|------|---------|-------------------|
| `MSG-W-BAT001-001` | 警告 | 「日替処理実行」ボタンクリック時の確認 | `{対象営業日} の日替処理を実行します。この操作は取り消せません。実行しますか？` |
| `MSG-S-BAT001-001` | 成功 | 日替処理が正常完了した場合 | `日替処理が完了しました（対象営業日: {対象営業日}）` |
| `MSG-E-BAT001-001` | エラー | 処理済みの営業日を指定して実行しようとした場合 | `{対象営業日} は既に処理済みです。同一営業日への二重実行はできません。` |
| `MSG-E-BAT001-002` | エラー | バッチ処理中にいずれかのステップが失敗した場合 | `日替処理が失敗しました。{失敗ステップ名} でエラーが発生しました。詳細は実行結果を確認してください。` |
| `MSG-I-BAT001-001` | 情報 | 再実行時に完了済みステップがスキップされた場合 | `ステップ{N}（{stepName}）は完了済みのためスキップしました` |
| `MSG-E-BAT001-003` | エラー | 対象営業日が未入力の場合 | `対象営業日は必須です。` |

## RPT-017 日次集計レポート出力ダイアログ

### 概要

BAT-002 の「日次集計レポート出力」ボタンから呼び出されるダイアログ。対象営業日を選択してPDF出力する。

### ダイアログ項目一覧

### レポート出力内容

| 項目 | 内容 |
|------|------|
| 対象営業日 | 選択した営業日 |
| 入荷件数 | 対象営業日の入庫完了件数（伝票ベース） |
| 入荷数量合計 | 対象営業日の入庫完了数量の合計 |
| 出荷件数 | 対象営業日の出荷完了件数（受注ベース） |
| 出荷数量合計 | 対象営業日の出荷完了数量の合計 |

| 未出荷件数 | 対象営業日時点で未完了の受注件数 |
