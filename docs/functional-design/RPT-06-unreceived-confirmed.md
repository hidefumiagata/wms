# レポート設計書 — 未入荷リスト（確定）

## 1. レポート概要

| 項目 | 内容 |
|------|------|
| **レポートID** | `RPT-06` |
| **レポート名** | 未入荷リスト（確定） |
| **用途** | 日替処理（バッチ）で確定されたスナップショットとして、営業日末時点の未入荷状況を振り返り・報告・エビデンスとして使用する。過去日の未入荷状況を遡って確認する際にも利用される |
| **対応API** | `API-RPT-006`（[API-10-report.md](API-10-report.md) を参照） |
| **用紙サイズ** | A4 |
| **向き** | 横（Landscape） |
| **呼び出し元画面** | INB-001（入荷一覧画面）から出力ボタンで呼び出し |

> 入力パラメータ（検索条件）は [API-10-report.md](API-10-report.md) の API-RPT-006 を参照。

## 2. PDFレイアウト

## 4. 特記事項

- **確定データ（スナップショット）**: 本レポートは日替処理（バッチ）で `unreceived_list_records` テーブルに保存された確定データを参照する。RPT-05（未入荷リスト（リアルタイム））とは異なり、出力タイミングに関わらず指定営業日末時点の状態が再現される。過去日のデータを遡って出力する用途にも適している。
- **営業日基準日の明示**: ヘッダーに「営業日基準日」を明示し、どの営業日時点のスナップショットであるかを明確にする。RPT-05のヘッダーが「基準日」であるのに対し、本レポートでは「営業日基準日（日替確定）」と表記して区別する。
- **遅延日数の算出**: API-RPT-006 のレスポンスには `delayDays` フィールドが含まれないため、PDF生成時に `batchBusinessDate - plannedDate` で算出する。
- **遅延日数の強調表示**: 遅延日数が7日以上の行は背景色をピンク（`#FFF1F1`）にして視覚的に強調する。
- **ソート順**: 仕入先名（昇順） → 入荷予定日（昇順。予定日が古い＝遅延が大きい順が上位）で出力する。データ取得仕様の詳細は API-RPT-006 の業務ロジックを参照。
- **データ0件の場合**: 指定した営業日の日替処理が未実行の場合、またはデータが存在しない場合は、明細テーブルの代わりに「該当する未入荷データはありません（指定営業日の日替処理が実行されていない可能性があります）」メッセージを中央表示する。
- **キャンセル・入庫完了済除外**: 日替処理時点で `STORED`（入庫完了）および `CANCELLED`（キャンセル）のステータスだった伝票はスナップショットに含まれない。対象ステータスの詳細は API-RPT-006 の業務ロジックを参照。
