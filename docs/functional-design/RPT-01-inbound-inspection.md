# レポート設計書 — 入荷検品レポート

## 1. レポート概要

| 項目 | 内容 |
|------|------|
| **レポートID** | `RPT-01` |
| **レポート名** | 入荷検品レポート |
| **用途** | 倉庫スタッフ・倉庫管理者が、入荷検品完了後に検品結果を確認・記録するために使用する。予定数と検品数の差異を品目ごとに把握し、仕入先への過不足連絡や入庫作業の判断材料とする |
| **対応API** | `API-RPT-001`（[API-10-report.md](API-10-report.md) を参照） |
| **用紙サイズ** | A4 |
| **向き** | 横（Landscape） |
| **呼び出し元画面** | INB-003（入荷予定詳細画面）から「検品レポート」ボタンで呼び出し |

> 入力パラメータ（検索条件）は [API-10-report.md](API-10-report.md) の API-RPT-001 を参照。

## 2. PDFレイアウト

### 2.1. ヘッダー（1ページ目のみ追加表示する情報）

共通ヘッダー（[_standard-report.md](_standard-report.md) 参照）に加え、以下の伝票情報を1ページ目に表示する。

| 要素 | 位置 | 内容 |
|------|------|------|
| 伝票番号 | 左 | `slipNumber`（例: `INB-2026-00123`） |
| 仕入先名 | 左 | `supplierName`（例: `テスト仕入先A`） |
| 入荷予定日 | 右 | `plannedDate`（例: `2026-03-14`） |

> これらの情報はAPIレスポンスの先頭行から取得する（1伝票=全行同一値）。

### 2.2. 明細テーブル カラム定義

| # | カラム名 | APIフィールド | 幅(mm) | 配置 | 書式 | 備考 |

| 2 | 商品コード | `productCode` | 28 | 左 | — | |
| 3 | 商品名 | `productName` | 50 | 左 | — | 長い場合は折り返し |
| 4 | ケース入数 | `casQuantity` | 18 | 右 | 整数 | |
| 5 | 予定数(ケース) | `plannedQuantityCas` | 20 | 右 | 整数 | |

| 7 | 差異(ケース) | `diffQuantityCas` | 18 | 右 | 整数 | 差異!=0 は太字 |

### 2.3. グルーピング・小計

グルーピングなし（フラットリスト）。1伝票の明細を行番号順に表示する。

### 2.4. 合計行（テーブル末尾）

| 表示位置 | 内容 |
|---------|------|
| 商品名列 | 「合計」ラベル |
| 予定数(ケース)列 | 全行の `plannedQuantityCas` 合計値 |
| 検品数(ケース)列 | 全行の `inspectedQuantityCas` 合計値 |
| 差異(ケース)列 | 全行の `diffQuantityCas` 合計値 |
| 予定数(バラ)列 | 全行の `plannedQuantityPcs` 合計値 |
| 検品数(バラ)列 | 全行の `inspectedQuantityPcs` 合計値 |
| 差異(バラ)列 | 全行の `diffQuantityPcs` 合計値 |

### 2.5. ページブレークルール

| ルール | 内容 |
|--------|------|
| 行数制限 | 25行/ページ（超過時に自動改ページ） |
| グループ改ページ | なし（グルーピングなし） |
| 孤立行防止 | 合計行が次ページに単独で送られる場合、直前の明細行とともに改ページする |

## 4. 特記事項

- **差異行の強調表示**: 差異数がゼロでない行（`diffQuantityPcs != 0` または `diffQuantityCas != 0`）は、行全体の背景色をピンク（`#FFF1F1`）で強調表示する。差異数の値自体は太字で表示する
- **ロット管理OFF商品**: 商品の `lot_manage_flag` が `false` の場合、ロット番号列は「—」（emダッシュ）で表示する
- **期限管理OFF商品**: 商品の `expiry_manage_flag` が `false` の場合、期限日列は「—」（emダッシュ）で表示する
- **データ0件**: 入荷伝票に明細が存在しない場合（通常は発生しない）、明細テーブル領域に「該当データがありません」メッセージを中央表示する
- **未検品明細**: 検品未実施（`inspected_qty` が NULL）の明細行は、検品数列・差異列を「—」で表示する
- **テンプレートファイル配置**: `backend/src/main/resources/templates/reports/RPT-01.html`
