## IF-001 ファイル一覧画面

### 1. 画面概要

| 項目 | 内容 |
|------|------|
| **画面ID** | `IF-001` |
| **画面名** | ファイル一覧 |
| **URL パス** | `/interface` |
| **対象ロール** | SYSTEM_ADMIN, WAREHOUSE_MANAGER |
| **概要** | Azure Blob Storage の `pending` フォルダに配置されたCSVファイルの一覧を表示する画面。I/F種別（入荷予定 / 受注）をタブで切り替え、各ファイルに対してバリデーション・取り込み操作を実行できる。ファイルの配置は外部システム（またはモック配置スクリプト）が行い、WMSユーザーはこの画面から取り込み操作のみを実施する。 |

### 4. イベント一覧

| イベントID | トリガー | 処理概要 | 遷移先 / 結果 | 実行可能ロール |
|----------|---------|---------|-------------|-------------|
| `EVT-IF001-001` | 画面初期表示 | `GET /api/v1/interface/{ifId}/files` を呼び出し（初期タブ: `ifId=IFX-001`）、入荷予定タブのpendingフォルダ内ファイル一覧を取得して表示する | 同画面（一覧表示） | SYSTEM_ADMIN, WAREHOUSE_MANAGER |
| `EVT-IF001-002` | I/F種別タブ切替 | 選択されたI/F種別に応じて `GET /api/v1/interface/{ifId}/files` を呼び出し、ファイル一覧を更新する | 同画面（一覧更新） | SYSTEM_ADMIN, WAREHOUSE_MANAGER |
| `EVT-IF001-003` | 「更新」ボタンクリック | 現在選択中のタブのI/F種別で `GET /api/v1/interface/{ifId}/files` を再呼び出しし、一覧を最新化する | 同画面（一覧更新） | SYSTEM_ADMIN, WAREHOUSE_MANAGER |
| `EVT-IF001-004` | 「バリデーション」ボタンクリック | `POST /api/v1/interface/{ifId}/validate` を呼び出し（リクエスト: `{fileName, warehouseId}`）、バリデーション結果をIF-002ダイアログで表示する。処理中はボタンを無効化しローディング表示する | IF-002ダイアログ表示 | SYSTEM_ADMIN, WAREHOUSE_MANAGER |
| `EVT-IF001-005` | 「取り込み」ボタンクリック | 取り込みモード選択の確認ダイアログ（MSG-W-IF001-001）を表示する。モード: 「全件取り込み」（バリデーション成功行のみ）/ 「全件破棄」 | 確認ダイアログ表示 | SYSTEM_ADMIN, WAREHOUSE_MANAGER |
| `EVT-IF001-006` | 取り込み確認ダイアログ「実行」クリック | `POST /api/v1/interface/{ifId}/import` を呼び出し（リクエスト: `{fileName, warehouseId, mode}`）、取り込みを実行する。処理中はローディング表示。完了後にMSG-S-IF001-001を表示し、ファイル一覧を再取得する | 同画面（一覧更新） | SYSTEM_ADMIN, WAREHOUSE_MANAGER |
| `EVT-IF001-007` | 「取り込み履歴へ」リンククリック | IF-003（取り込み履歴画面）へ画面遷移する | `/interface/history` へ遷移 | SYSTEM_ADMIN, WAREHOUSE_MANAGER |

### 5. メッセージ一覧

| メッセージID | 種別 | 発生条件 | メッセージ文（日本語） |
|------------|------|---------|-------------------|
| `MSG-W-IF001-001` | 警告 | 「取り込み」ボタンクリック時の確認 | `{ファイル名} を取り込みます。取り込みモードを選択してください。` |
| `MSG-S-IF001-001` | 成功 | 取り込み完了（SUCCESS_ONLYモード） | `{ファイル名} の取り込みが完了しました。（成功: {成功件数}件 / エラー: {エラー件数}件）` |
| `MSG-S-IF001-002` | 成功 | 取り込み完了（DISCARDモード） | `{ファイル名} を破棄しました。ファイルはprocessedフォルダへ移動されました。` |
| `MSG-E-IF001-001` | エラー | ファイル一覧取得失敗（Blob Storage接続エラー） | `ファイル一覧の取得に失敗しました。しばらく経ってから再試行してください。` |
| `MSG-E-IF001-002` | エラー | バリデーション実行失敗（Blob Storage接続エラー等） | `バリデーションの実行に失敗しました。しばらく経ってから再試行してください。` |
| `MSG-E-IF001-003` | エラー | 取り込み実行失敗（DB登録エラー等） | `取り込み処理に失敗しました。しばらく経ってから再試行してください。` |
| `MSG-E-IF001-004` | エラー | ファイルサイズが50MBを超過 | `ファイルサイズが上限（50MB）を超えています。` |
| `MSG-I-IF001-001` | 情報 | ファイル一覧が0件 | `取り込み待ちのファイルはありません。` |

**表示方法の補足**:
- `MSG-W-IF001-001`: `el-message-box`（confirm ダイアログ）として表示。ダイアログ内にモード選択ラジオ（「成功行のみ取り込む」/「全件破棄」）を配置する
- `MSG-S-IF001-001`, `MSG-S-IF001-002`: `el-message`（type: success）でトースト通知
- `MSG-E-IF001-001` 〜 `MSG-E-IF001-004`: `el-message`（type: error）でトースト通知
- `MSG-I-IF001-001`: テーブル内の空状態として `el-empty` コンポーネントで表示
