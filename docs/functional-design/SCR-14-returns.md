# 機能設計書 — 画面設計 返品管理（RTN-001）

---

## RTN-001 返品登録画面

### 1. 画面概要

| 項目 | 内容 |
|------|------|
| **画面ID** | `RTN-001` |
| **画面名** | 返品登録 |
| **URL パス** | `/returns/new` |
| **対象ロール** | WAREHOUSE_MANAGER, WAREHOUSE_STAFF |
| **概要** | 仕入先への返品伝票を登録する。1伝票1商品。返品種別（入荷返品・在庫返品・出荷返品）を選択し、商品・数量・荷姿・返品理由を入力して登録する。在庫返品の場合は在庫から即時減算される。業務ルールの詳細は [functional-requirements/08-returns.md](../functional-requirements/08-returns.md) を参照。 |

**コンテンツエリア詳細**

#### 基本情報セクション

| 項目 | 配置 |
|------|------|
| 返品種別（セレクト） | 1行目 |
| 商品コード（テキスト + 検索ボタン） | 2行目 |
| 商品名（読み取り専用） | 3行目 |
| 数量（数値入力） | 4行目左 |
| 荷姿（セレクト） | 4行目右 |

#### 在庫返品セクション（条件付き表示）

返品種別が「在庫返品」の場合のみ表示される。

| 項目 | 配置 |
|------|------|
| ロケーション（セレクト） | 1行目 |
| 現在在庫数（読み取り専用） | 2行目左 |
| 有効在庫数（読み取り専用） | 2行目右 |

#### 取引先・理由セクション

| 項目 | 配置 |
|------|------|
| 仕入先（テキスト + 検索ボタン） | 1行目 |
| 返品理由（セレクト） | 2行目 |
| 返品理由備考（テキストエリア） | 3行目 |

#### 条件付き表示項目セクション

| 項目 | 表示条件 |
|------|---------|
| 関連伝票番号 | 返品種別が「入荷返品」または「出荷返品」の場合 |
| ロット番号 | 商品マスタの `lot_manage_flag = true` の場合 |
| 賞味期限 | 商品マスタの `expiry_manage_flag = true` の場合 |

### 4. イベント一覧

| イベントID | トリガー | 処理概要 | 遷移先 / 結果 | 実行可能ロール |

| `EVT-RTN001-001` | 画面初期表示 | クエリパラメータを確認し、`returnType` があれば返品種別をプリセット、`relatedSlipNumber` があれば関連伝票番号をプリセット（入荷検品画面からの遷移時） | 同画面（初期値設定） | WAREHOUSE_MANAGER, WAREHOUSE_STAFF |
| `EVT-RTN001-002` | 商品検索ボタンクリック / 商品コード入力後Enter | API `GET /api/v1/master/products?keyword={keyword}` で商品を検索し、検索結果ダイアログを表示。1件の場合は直接選択 | 同画面（検索ダイアログ表示） | WAREHOUSE_MANAGER, WAREHOUSE_STAFF |
| `EVT-RTN001-003` | 商品選択（検索結果から選択） | 商品名を自動表示。商品マスタの `lot_manage_flag`・`expiry_manage_flag` を判定してロット番号・賞味期限フィールドの表示を制御。商品に紐づく仕入先を自動候補表示。在庫返品時はロケーション候補をリセット | 同画面（フィールド表示制御・仕入先候補表示） | WAREHOUSE_MANAGER, WAREHOUSE_STAFF |
| `EVT-RTN001-004` | 返品種別変更 | 返品種別に応じた条件付き表示を制御。「在庫返品」→ロケーション選択を表示。「入荷返品」「出荷返品」→関連伝票番号を表示。他の種別固有項目は非表示に戻す | 同画面（フィールド表示切替） | WAREHOUSE_MANAGER, WAREHOUSE_STAFF |
| `EVT-RTN001-005` | 在庫返品時: 商品・荷姿選択確定後 | API `GET /api/v1/inventory?productId={productId}&warehouseId={warehouseId}` で在庫のあるロケーション候補を取得し、ロケーションセレクトの選択肢を動的更新 | 同画面（ロケーションセレクト更新） | WAREHOUSE_MANAGER, WAREHOUSE_STAFF |
| `EVT-RTN001-006` | 在庫返品時: ロケーション選択変更 | 選択されたロケーション×商品×荷姿の現在在庫数・有効在庫数を表示 | 同画面（在庫数表示更新） | WAREHOUSE_MANAGER, WAREHOUSE_STAFF |
| `EVT-RTN001-007` | [登録] ボタンクリック | フォームバリデーション実行（必須チェック・数量チェック・在庫返品固有チェック）→ 確認ダイアログ表示（「{商品名} を {数量}{荷姿} 返品登録しますか？」） | 確認ダイアログ表示 | WAREHOUSE_MANAGER, WAREHOUSE_STAFF |
| `EVT-RTN001-008` | 確認ダイアログ [OK] クリック | API `POST /api/v1/returns` を呼び出して返品伝票を登録 | 成功時: 成功メッセージ表示、フォームクリア | WAREHOUSE_MANAGER, WAREHOUSE_STAFF |
| `EVT-RTN001-009` | [キャンセル] ボタンクリック | 入力内容をクリアして初期状態に戻す | 同画面（フォームクリア） | WAREHOUSE_MANAGER, WAREHOUSE_STAFF |

| `MSG-S-RTN001-001` | 成功 | 返品登録完了 | 返品伝票 {伝票番号} を登録しました。 |
| `MSG-E-RTN001-001` | エラー | 返品種別が未選択 | 返品種別は必須です。 |
| `MSG-E-RTN001-002` | エラー | 商品コードが未入力 | 商品コードは必須です。 |
| `MSG-E-RTN001-003` | エラー | 商品コードに該当する商品が存在しない | 指定された商品コードが見つかりません。 |
| `MSG-E-RTN001-004` | エラー | 数量が未入力または0以下 | 数量は1以上の整数を入力してください。 |
| `MSG-E-RTN001-005` | エラー | 荷姿が未選択 | 荷姿は必須です。 |
| `MSG-E-RTN001-006` | エラー | 在庫返品時にロケーションが未選択 | 在庫返品の場合、ロケーションは必須です。 |
| `MSG-E-RTN001-007` | エラー | 在庫返品時に返品数量が有効在庫数を超過 | 返品数量が有効在庫数を超えています。（有効在庫数: {有効在庫数}{荷姿}） |
| `MSG-E-RTN001-008` | エラー | 在庫返品時に引当済み在庫が存在 | 引当済みの在庫があるため返品できません。（引当数: {引当数}{荷姿}） |
| `MSG-E-RTN001-009` | エラー | 在庫返品時に棚卸ロック中のロケーション | 指定のロケーションは棚卸中のため返品できません。 |
| `MSG-E-RTN001-010` | エラー | 返品理由が未選択 | 返品理由は必須です。 |
| `MSG-E-RTN001-011` | エラー | 「その他」理由選択時に備考が未入力 | 返品理由が「その他」の場合、備考は必須です。 |
| `MSG-E-RTN001-012` | エラー | 返品理由備考が500文字超過 | 返品理由備考は500文字以内で入力してください。 |
| `MSG-E-RTN001-013` | エラー | 登録API呼び出し失敗 | 返品伝票の登録に失敗しました。時間をおいて再試行してください。 |
| `MSG-I-RTN001-001` | 情報 | 商品検索結果が0件 | 検索条件に一致する商品がありません。 |
| `MSG-I-RTN001-002` | 情報 | 在庫返品時にロケーション候補が0件 | 選択した商品・荷姿の在庫があるロケーションが見つかりません。 |

**表示方法の補足**:
- `MSG-E-RTN001-001` 〜 `MSG-E-RTN001-006`, `MSG-E-RTN001-010` 〜 `MSG-E-RTN001-012`: フォーム項目直下にインラインエラーとして表示
- `MSG-E-RTN001-007` 〜 `MSG-E-RTN001-009`, `MSG-E-RTN001-013`: フォーム上部にエラーバナー（`alert-error` スタイル）として表示
- `MSG-W-RTN001-001`: `el-message-box`（confirm ダイアログ）として表示
- `MSG-S-RTN001-001`: `el-message`（type: success）でトースト通知
- `MSG-I-RTN001-001`, `MSG-I-RTN001-002`: `el-message`（type: info）でトースト通知
