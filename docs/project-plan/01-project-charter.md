# プロジェクト憲章 (Project Charter)

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| **プロジェクト名** | WMS ShowCase - Claude Code による本格業務システム開発 |
| **作成日** | 2026-03-12 |
| **バージョン** | 1.0 |

## 2. プロジェクト目的

| # | 目的 |
|---|------|
| 1 | 汎用WMSパッケージソフトウェアの構築（業種・規模を問わず運用可能） |
| 2 | Claude Code ShowCase（Claude Codeで本格業務システムが作れることの実証） |

## 3. 機能スコープ

### 対象機能（In Scope）

| カテゴリ | 機能 |
|---------|------|
| **入荷管理** | 入荷予定取り込み・入荷検品・入庫指示 |
| **在庫管理** | ロケーション管理・棚卸・在庫照会 |
| **出荷管理** | 受注取り込み・ピッキング指示・出荷検品・配送指示 |
| **マスタ管理** | 商品・取引先・ロケーション・ユーザ |
| **レポート** | 入荷予定・入庫実績・未入荷リスト・在庫一覧・在庫推移・在庫訂正一覧・棚卸リスト・棚卸結果・ピッキング指示書・出荷検品・配送リスト・未出荷リスト・日次集計（全13種、詳細は機能要件定義書参照） |
| **外部連携I/F** | 入荷予定取り込みI/F・受注取り込みI/F |
| **バッチ処理** | 日替処理（営業日更新・入荷実績集計・出荷実績集計・在庫集計・トランデータバックアップ） |

### 対象外機能（Out of Scope）

| カテゴリ | 理由 |
|---------|------|
| **外部設備連携** | バーコードリーダー・RFID・自動倉庫等は対象外 |

## 5. 制約条件 & 前提条件

| 項目 | 内容 |
|------|------|
| **クラウド** | Azure |
| **フロントエンド** | Vue 3 + TypeScript + Element Plus + Vite + Pinia + vue-i18n（日英対応） |
| **バックエンド** | Spring Boot 3.x + Java 21 + Gradle |
| **データベース** | Azure Database for PostgreSQL Flexible Server（B1ms） |
| **コンテナ** | Azure Container Apps（min replicas=0） |
| **ストレージ** | Azure Blob Storage（フロントエンド静的ホスティング + I/Fファイル） |
| **IaC** | Terraform |
| **認証** | Spring Security + JWT + httpOnly Cookie |
| **開発期間** | 約1週間（Claude Code並列開発でアグレッシブに進める） |
| **コスト** | 極力ゼロ（Azureの無料枠・低コストプラン、OSSのみ使用を原則とする） |

> **注記**：1週間でフル機能WMSを完成させるのは通常不可能だが、Claude Codeのマルチエージェント並列開発をフル活用することでこれを実現することがShowCaseの核心部分である。

## 6. 成功基準（Definition of Done）

| レベル | 内容 | 実現方式 |
|--------|------|---------|
| **A** | 全機能が画面から操作できる（テストデータあり） | 完全実装 |
| **B** | 外部連携I/Fによるファイル取り込み | モック実装 |
| **C** | バッチ処理の実行 | 手動実行（スケジューリングなし） |
| **D** | 本番同等データ量での動作 | 完全実装（パフォーマンステスト含む） |
