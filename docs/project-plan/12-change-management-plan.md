# 変更管理計画 (Change Management Plan)

## 変更分類と承認プロセス

| 変更種別 | 定義 | プロセス |
|---------|------|---------|
| **軽微な変更** | 画面レイアウト・項目追加・バリデーション変更など | 議論 → 合意 → ドキュメント即更新 |
| **重大な変更** | スコープ追加・削除、成功基準の変更、アーキテクチャ変更 | 議論 → 合意 → プロジェクト憲章・WBS・関連計画書を合わせて更新 |

## 変更履歴管理

- すべての変更はGitのコミット履歴で追跡する
- 重大な変更はGitHub Issueに記録し、機能IDで紐付ける
