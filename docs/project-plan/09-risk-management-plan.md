# リスク管理計画 (Risk Management Plan)

## リスク一覧

| ID | リスク | 影響 | 対応策 |
|----|--------|------|--------|
| R1 | 技術スタック選定の失敗（無料枠に収まらない等） | コスト超過 | アーキテクチャ設計時に事前検証 |
| R2 | Claude Codeのコンテキスト切れによる開発中断 | 作業ロス | グランドルール通り記録保存して新セッションで継続 |
| R3 | スコープクリープ（機能が膨らむ） | 完成しない | 変更管理計画に従い都度承認 |
| R4 | Azureの無料枠・低コストプラン超過 | 予期せぬコスト | 設計段階でリソース試算、Azureコストアラート設定 |
| R5 | 1週間で完成しない | ShowCase未達 | MVP優先、機能は段階的リリース |
| R6 | GitHubへの誤ったコードや認証情報の公開 | セキュリティ事故 | .gitignore徹底、シークレット管理ルール徹底 |
