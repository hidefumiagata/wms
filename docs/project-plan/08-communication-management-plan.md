# コミュニケーション管理計画 (Communication Management Plan)

## 1. ドキュメント管理

| 項目 | 方針 |
|------|------|
| **言語** | 日本語で作成し、後から英語翻訳して両方保管 |
| **保管場所** | `docs/` フォルダ（Markdown形式） |
| **ファイル命名** | 日本語版：`xxx.md`、英語版：`xxx.en.md` |

## 2. 図・ダイアグラムのフォーマット

| 種別 | フォーマット | 用途 |
|------|------------|------|
| **構成図** | draw.io（`.drawio`） | システム構成図・インフラ図・アーキテクチャ図 |
| **フローチャート** | Mermaid（Markdownに埋め込み） | 業務フロー・処理フロー・シーケンス図 |

## 3. コミットメッセージ

日英併記で記述する。

```
feat: ユーザマスタ登録機能を追加 / Add user master registration feature

- ユーザ登録・編集・削除APIを実装 / Implement user CRUD API
- ユーザ一覧画面を実装 / Implement user list screen
```

## 4. Pull Request

| 項目 | 方針 |
|------|------|
| **タイトル** | 日英併記 |
| **本文** | テスト結果・レビュー指摘内容を記載（Claude Codeが作成） |
| **承認者** | プロジェクトオーナー |

## 5. Issue管理

| 項目 | 方針 |
|------|------|
| **管理ツール** | GitHub Issues |
| **言語** | 日本語 |
| **紐付け** | 機能ID（例：WMS-INB-001）をタイトルに含める |
